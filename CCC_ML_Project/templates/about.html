Section 01 · Objective

Investigation Objective

The goal was not to count crime. It was to discover which cities share the same behavioral fingerprint — using unsupervised machine learning to surface structure the data already contained.

01 / Pattern Discovery

Find What Was Hidden

Raw crime numbers carry no structure on their own. The investigation aimed to group cities not by geography or politics, but by the underlying behavioral patterns embedded in five key metrics. The algorithm was given the numbers and asked a single question: who belongs together?

02 / Crime Intelligence

Structure Over Statistics

Traditional dashboards report what happened. Crime Atlas was built to answer a more important question: which cities share the same crime identity? That distinction shifts the frame entirely — from incident reporting to behavioral classification.

03 / Data-Driven Analysis

The Machine Decides

No categories were invented in advance. No assumptions were built in. An unsupervised algorithm was given the data and allowed to organize it on its own terms. The resulting clusters were not assigned — they were discovered.

Section 03 · Processing

Evidence Processing Pipeline

Raw evidence cannot be analyzed without first being processed. Every record was cleaned, verified, scaled, and prepared before the model was ever engaged.

STEP 01

Data Collection

Raw crime records loaded from structured CSV format. 40,000+ incident-level records spanning 50 Indian cities, each containing time, location, victim data, and case status.

STEP 02

Cleaning

Duplicate entries removed. Inconsistent fields corrected. Records with corrupted or implausible values flagged and excluded from the final analysis dataset.

STEP 03

Missing Values

Missing numerical values imputed using column means. Missing categorical values handled via mode imputation. No record was discarded for a single missing field.

STEP 04

Feature Selection

Five analytically significant features extracted by aggregating incident records to city level. Features were chosen for behavioral relevance, not statistical convenience.

STEP 05

StandardScaler

All five features normalized using StandardScaler. Each variable transformed to zero mean and unit variance — ensuring no single feature dominated by scale alone.

Section 04 · Analysis Engine

The Analysis Engine

With evidence secured and processed, the investigation required a classification engine capable of finding structure without being told what to look for.

Algorithm · K-Means Clustering

K-Means Clustering

K-Means is an unsupervised learning algorithm that partitions data into K distinct clusters by minimizing the variance within each group. It iterates — reassigning data points and repositioning cluster centroids — until the assignments stabilize.

Unlike supervised models, K-Means requires no labeled training data. It was given 50 cities and five behavioral measurements, and was asked to organize them by similarity. It did not know what crime clusters looked like. It found out.

kmeans_engine.py
from sklearn.cluster import KMeans
model = KMeans(n_clusters=4, random_state=42)
labels = model.fit_predict(X_scaled)
✓ Converged in 14 iterations
✓ 4 clusters identified
Why K-Means Was Selected
01

No Labels Existed

There were no pre-defined crime categories to train against. Unsupervised learning was the only valid approach — the algorithm had to find the categories itself.

02

City-Level Grouping

K-Means is purpose-built for partitioning compact, well-separated groups in continuous feature space — exactly the structure expected across distinct urban crime profiles.

03

Interpretability

Each cluster centroid provides a readable behavioral profile — a concrete set of average values that define what it means to belong to that group.

04

Scalability

K-Means scales efficiently to larger datasets. The same engine that analyzed 50 cities can be retrained on 500 — without architectural changes.

Section 05 · Visualization

Pattern Visualization

Five-dimensional data cannot be shown on a screen. Principal Component Analysis was the lens that made the invisible structure visible.

PCA Scatter — Principal Components 1 & 2
What is PCA?

Dimensionality Reduction

Principal Component Analysis compresses multi-dimensional data into a smaller number of components that capture the most variance. Five behavioral features were reduced to two principal components — enough to plot every city on a 2D plane.

Why It Was Used

Making Clusters Visible

The K-Means algorithm works in five dimensions simultaneously — invisible to the human eye. PCA created a projection where cluster separation could be seen directly, validating that the groupings were real and not statistical artifacts.

What It Revealed

Four Distinct Regions

When the 50 cities were plotted using the first two principal components, four visually separable regions emerged — each corresponding precisely to one of the K-Means behavioral clusters. The data had always been structured this way.

Section 08 · Obstacles

Challenges Faced

No investigation proceeds without resistance. These are the obstacles encountered during the development of Crime Atlas and how they were resolved.

Obstacle 01

Selecting Meaningful Features

With eight available columns in the raw dataset, choosing the five that would produce analytically meaningful clusters required careful domain reasoning. Features had to be behaviorally relevant — not just statistically available. Multiple feature combinations were tested before the final five were confirmed.

Obstacle 02

Determining Optimal Cluster Count

K-Means requires the number of clusters to be specified in advance — but the data does not tell you what that number should be. The Elbow Method and Silhouette Score were both applied independently. Four clusters was the only value where both diagnostics converged.

Obstacle 03

Interpreting Unsupervised Results

Cluster labels produced by K-Means are arbitrary integers — 0, 1, 2, 3. Transforming them into meaningful behavioral profiles required manual inspection of each cluster's centroid values. The names — High Crime, Moderate, Low Crime, Outlier — emerged from that analysis, not from the algorithm.

Obstacle 04

Visualizing High-Dimensional Patterns

Plotting five-dimensional data requires projection. PCA was chosen for its ability to preserve the maximum amount of variance in two dimensions — but any projection involves information loss. Verifying that the 2D scatter faithfully represented the true cluster structure required cross-validation against the original feature space.

Connect Crime Atlas to live crime data APIs to continuously update city-level behavioral profiles as new incidents are recorded — turning a static analysis into a dynamic intelligence system.

Track how city cluster assignments shift over time. A city moving from Cluster C to Cluster A is a signal — temporal analysis would detect those transitions before they become crises.

Apply autoencoders or variational models to learn richer, non-linear representations of city crime profiles — potentially surfacing finer-grained cluster structure invisible to K-Means.

Scale from 50 cities to 500. Add socioeconomic features — literacy rates, population density, unemployment. Richer inputs produce richer behavioral profiles.
